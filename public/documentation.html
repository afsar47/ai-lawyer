Introduction

Welcome to AI Lawyer, an AI-powered legal practice management system designed to streamline daily operations for law firms and legal professionals. The platform combines core practice management (clients, cases, tasks, deadlines, hearings, documents, billing) with AI-assisted tools for drafting, contract review, matter intake, and legal research.

Key Benefits:

  • Streamlined client and case management
  • Calendar, appointments, and hearing scheduling
  • AI-powered legal drafting, contract review, and knowledge search
  • Document management with templates and uploads
  • Secure data management with role-based access and hashed passwords

Important Disclaimer – OpenAI API Required for AI Features

  • External API key required: An OpenAI API key is required for the AI features to function.
  • Buyer responsibility for costs: Any costs associated with the use of the OpenAI API are the sole responsibility of the buyer.
  • No pre-paid subscription included: This item does NOT include a pre-paid OpenAI API subscription. You must purchase API credits directly from OpenAI.

Core features (case management, billing, documents, etc.) work without an AI API key.

Installation

Downloading from CodeCanyon

This script is available on CodeCanyon. After purchase, you'll receive:

  1. main_files.zip - The initial download package
  2. After extracting main_files.zip, you'll get ai-lawyer.zip - This is the actual script file
  3. Extract ai-lawyer.zip to begin installation

Prerequisites

1. Install Node.js

Download and install Node.js from the official website:

Download: https://nodejs.org/

Choose the LTS (Long Term Support) version for stability.

Verify installation:

node --version
npm --version

2. Install MongoDB

Choose one of the following options:

Option A: MongoDB Atlas (Cloud - Recommended)
  • Sign up at MongoDB Atlas
  • Create a free cluster (M0 tier available)
  • Get your connection string
Option B: Local MongoDB Installation
  • Download from MongoDB Community Server
  • Install and start MongoDB service
  • Default connection: mongodb://localhost:27017

Installation Steps

Step 1: Download and Extract Files

Download the script from CodeCanyon:

  1. Log in to your CodeCanyon account
  2. Navigate to your downloads section
  3. Download main_files.zip
  4. Extract main_files.zip to get ai-lawyer.zip
  5. Extract ai-lawyer.zip to your desired location

Note: After extraction, you should have a folder named ai-lawyer (or similar) containing all the project files.

Navigate to the extracted folder:

cd ai-lawyer

Step 2: Install Dependencies

Navigate to the project directory and install all required packages:

npm install

This will install all dependencies listed in package.json. This process may take a few minutes.

Step 3: Configure Environment Variables

Create a .env.local file in the root directory with the following variables:

MONGODB_URI=mongodb://localhost:27017/ai-lawyer
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=your-secret-key-here

Environment Variables Explained:

  • MONGODB_URI: Your MongoDB connection string. For MongoDB Atlas, use: mongodb+srv://username:password@cluster.mongodb.net/ai-lawyer
  • NEXTAUTH_URL: The base URL of your application (use your production domain in production)
  • NEXTAUTH_SECRET: A random secret key for encrypting JWT tokens. Generate one using: openssl rand -base64 32

Important:

  • Keep your .env.local file secure and never share it
  • GPT-4.1 API key can be configured later through the Settings → AI Settings page

Step 4: Set Up Database

Ensure MongoDB is running and accessible:

For Local MongoDB:
# Start MongoDB service
# Windows: net start MongoDB
# macOS: brew services start mongodb-community
# Linux: sudo systemctl start mongod
For MongoDB Atlas:
  • Ensure your IP address is whitelisted in Atlas Network Access
  • Create a database user with read/write permissions
  • Use the connection string provided by Atlas

Step 5: Seed Database (Optional)

Populate the database with sample data for testing:

npm run seed

This creates sample clients, cases, and data for demonstration purposes.

Step 6: Start Development Server

Run the application in development mode:

npm run dev

The application will start on http://localhost:3000

Success! You should see:

✓ Ready in X seconds
○ Local: http://localhost:3000

Troubleshooting

Common Issues and Solutions

Issue: MongoDB Connection Failed
  • Verify MongoDB is running: mongosh or check service status
  • Check connection string in .env.local
  • For Atlas: Verify IP whitelist and credentials
  • Check firewall settings
Issue: Port 3000 Already in Use
  • Stop other applications using port 3000
  • Or use a different port: PORT=3001 npm run dev
Issue: Module Not Found Errors
  • Delete node_modules folder
  • Delete package-lock.json
  • Run npm install again
Issue: Build Errors
  • Clear Next.js cache: rm -rf .next
  • Check TypeScript errors: npm run lint
  • Ensure all environment variables are set
Issue: AI Features Not Working
  • Verify GPT-4.1 API key is correctly configured
  • Check API key permissions and quotas
  • Test API connection in Settings → AI Settings
  • Check browser console for error messages

AI-Powered Features

AI Lawyer includes a suite of AI-powered tools designed to enhance legal workflows: drafting, contract review, matter intake, motion drafting, letter packs, and legal knowledge search. All AI features are powered by GPT-4.1 (configure your API key in Settings → AI Settings).

AI features available:

  • AI Assistant — general legal queries and context-aware help
  • AI Drafting — draft documents with AI
  • AI Matter Intake — intake new matters with structured AI assistance
  • AI Timeline — build case timelines
  • AI Contract Review — review contracts and flag issues
  • AI Motion Draft — draft motions with AI
  • AI Letter Pack — generate letter packs
  • AI Knowledge Search — search legal knowledge bases

AI Assistant

Your intelligent legal assistant for queries about law, procedures, case context, and practice management. It can reference your matters and documents when configured.

Key Features:

  • Legal research and procedural questions
  • Case and client context when relevant
  • Drafting and citation suggestions
  • Best-practice and workflow tips

Example Queries:

  • "What are the elements of a breach of contract claim?"
  • "Summarize the key deadlines for case X"
  • "Draft a short engagement letter for a new client"

AI Drafting

Use AI to draft legal documents, letters, and clauses. Provide context (e.g., matter type, parties, key terms) and refine the output.

Features:

  • Draft from prompts and templates
  • Edit and iterate on AI-generated text
  • Link drafts to cases and matters

AI Matter Intake

Streamline new matter intake with AI. Capture matter type, parties, key facts, and conflicts information in a structured way and create matters or link to existing cases.

Features:

  • Guided intake with AI-suggested fields
  • Conflict and eligibility checks
  • Create case and client records from intake

AI Timeline

Build and visualize case timelines from facts, documents, or narrative. AI helps order events and highlight key dates for deadlines and strategy.

Features:

  • Generate timeline from case description or uploaded content
  • Edit and reorder events
  • Export or link to case deadlines

AI Contract Review

Upload or paste contract text for AI-assisted review. Get summaries, risk flags, and suggested edits. Use as support only—always apply professional judgment.

Features:

  • Summary of key terms and obligations
  • Risk and anomaly highlighting
  • Suggested amendments or questions for counterparty

AI Motion Draft

Draft court motions with AI. Provide case context, relief sought, and jurisdiction; refine the draft to match your style and local rules.

Features:

  • Structured motion drafting (caption, facts, law, prayer)
  • Citation and authority suggestions
  • Edit and export for filing

AI Letter Pack

Generate sets of letters (e.g., client letters, opposing counsel, courts) from matter details. Customize templates and merge with case data.

Features:

  • Generate multiple letters from one matter input
  • Consistent formatting and tone
  • Edit and export individually or as a pack

AI Knowledge Search

Search across your legal knowledge base (policies, precedents, memos) using natural language. Get relevant excerpts and source references.

Features:

  • Natural language queries over uploaded or linked content
  • Cited snippets and document links
  • Use for research and consistency across matters

Tips & Best Practices

General Tips

  • Keep client and case information up-to-date for accurate AI outputs
  • Use search to quickly find clients, cases, documents, or appointments
  • Review deadlines and hearings regularly; use the calendar view
  • Export important data and reports for backup
  • Use AI features as support tools; always apply professional legal judgment

AI Features Best Practices

  • Verify AI-generated drafts and contract reviews before use or filing
  • Use matter intake to keep matter data consistent for AI tools
  • Ensure your GPT-4.1 API key is valid and has sufficient credits (Settings → AI Settings)
  • Review and redact confidential information before pasting into AI

Security & Privacy

  • Keep your password secure and change it regularly

  • Review access logs and ensure secure connections when accessing the system

Efficiency Tips
